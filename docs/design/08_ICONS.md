# 08 Icons

## Icon System

Icons should support clarity, not ornamentation. The icon style must be minimal, precise, and aligned with the brand's calm premium tone.

## Sizes

- 12 px: fine detail, metadata, and compact labels
- 16 px: common interface size
- 20 px: navigation and secondary actions
- 24 px: key features and primary UI emphasis
- 32 px: occasional feature illustrations

## Stroke Width

- Use a consistent stroke width across the system.
- Recommended default: balanced medium stroke with optical adjustments for small sizes.
- Do not mix heavy and thin icon styles.

## Usage

- Use icons to improve scanning and orientation.
- Pair icons with labels where meaning could be ambiguous.
- Use sparingly in editorial areas so typography remains dominant.
- Avoid decorative icon clusters or animated icon flourishes.

## Icon Rules

- Keep icons geometric and calm.
- Avoid cartoonish or filled icon families unless a specific system requires them.
- Icons must be clear in both light and dark mode.
- Do not use icons to replace meaningful copy.
