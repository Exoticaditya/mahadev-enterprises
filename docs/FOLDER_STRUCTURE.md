# Folder Structure

## Proposed Structure

- app/
- components/
- content/
- data/
- lib/
- styles/
- public/
- docs/

## App Layer

Use route groups to separate marketing pages, utility pages, and SEO landing pages while keeping the public site simple to navigate.

## Components Layer

Split components by responsibility: layout, sections, forms, product, motion, and UI primitives.

## Content Layer

Store copy, SEO metadata, and structured page content separately so messaging can be updated without rewriting component logic.

## Data Layer

Use curated data modules for products, collections, testimonials, FAQs, and SEO targets.

## Library Layer

Keep shared helpers for formatting, metadata, image handling, and enquiry payload preparation.

## Design Rule

The file structure should reflect the user experience. Navigation, content, and conversion logic should be easy to locate and maintain.
