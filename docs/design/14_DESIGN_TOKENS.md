# 14 Design Tokens

## Token Philosophy

Everything important should be tokenized so the system stays consistent as the site grows. Tokens are the contract between design intent and implementation discipline.

## Token Categories

### Color Tokens

- Brand core
- Surface core
- Neutral scale
- Semantic states
- Dark mode palette
- Glass overlays
- Gradient stops
- Shadow colors

### Typography Tokens

- Font families
- Font sizes
- Font weights
- Line heights
- Letter spacing
- Display scale
- Body scale
- Button scale
- Label scale

### Spacing Tokens

- Micro spacing
- Component spacing
- Section spacing
- Layout spacing
- Container padding

### Radius Tokens

- Small radius for controls
- Medium radius for cards and chips
- Large radius for media or hero treatments

### Shadow Tokens

- Subtle shadow
- Elevated shadow
- Dark mode shadow
- Glass shadow

### Motion Tokens

- Fast duration
- Standard duration
- Slow duration
- Hover duration
- Reveal duration
- Stagger duration
- Easing curves

### Layout Tokens

- Container max widths
- Section gutters
- Breakpoint thresholds
- Grid column systems
- Vertical rhythm values

### Interaction Tokens

- Focus ring
- Hover elevation
- Disabled opacity
- Loading feedback
- Transition consistency

## Token Rules

- Tokens must be descriptive and reusable.
- One-off values should be avoided unless they solve a clearly unique case.
- The system should stay visually stable even as content scales.
- A token should exist only if it protects consistency or quality.
