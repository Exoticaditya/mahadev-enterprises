# 11 Component Guidelines

## Component Philosophy

Components must be consistent, reusable, and unmistakably part of the same brand system. Every component should support orientation, trust, or conversion.

## Navbar

- Should feel light, structured, and premium.
- Keep the number of top-level items small.
- Include a prominent enquiry path without making it noisy.
- Maintain strong state clarity on scroll and route changes.

## Footer

- Should act as a trust and navigation anchor.
- Include useful links, business context, and contact clarity.
- Avoid cluttered link dumps.

## Forms

- Forms must be concise and reassuring.
- Only ask for what is needed to start a useful conversation.
- Labels, helper text, and validation must be clear and calm.

## Sections

- Every section should have a clear purpose.
- Sections should maintain visual rhythm across pages.
- Avoid repetitive layouts unless repetition serves clarity.

## CTA

- CTAs should be elegant and decisive.
- Use the same action language across the site where possible.
- Avoid competing CTA hierarchies in a single viewport.

## Product Grid

- Should be highly scannable.
- Maintain consistent tile heights or predictable rhythm.
- Cards must never feel cramped or crowded.

## Gallery

- Use as a visual proof layer.
- Keep captions minimal and useful.
- Do not turn galleries into noisy collage walls.

## Quote Form

- The quote or enquiry form should look like a premium service path, not a generic contact block.
- Contextual cues should explain what the user will receive.
- Friction should stay low and confidence high.

## Blog

- Blog layouts should feel editorial and refined.
- Visual hierarchy should support reading rather than shouting for attention.
- Articles should feel authoritative and calm.

## Testimonials

- Testimonials should be short, credible, and specific.
- Prefer proof over praise.
- Layout should make the testimonial feel like evidence, not marketing copy.

## Component Rules

- No component should introduce a new style without a reason.
- Reuse patterns across pages to reinforce brand memory.
- Every component must degrade gracefully on mobile.
