# 04 Buttons

## Button Philosophy

Buttons should feel confident, restrained, and intentional. They are the conversion mechanism of a premium site, so they must be visually clear without becoming loud.

## Primary Button

- Used for the main conversion action on a page.
- Should be the highest visual priority after the headline.
- Recommended label examples: Enquire Now, Request Catalogue, Talk to Us.
- Should appear solid and materially grounded.

## Secondary Button

- Used for a supporting but still important action.
- Often pairs with the primary action in hero areas and CTA sections.
- Should feel lighter than the primary button but still clearly interactive.

## Ghost Button

- Used for low-pressure actions such as view details or learn more.
- Should maintain structure with minimal visual weight.
- Best used where multiple choices appear together.

## Icon Button

- Used for sliders, navigation, quick actions, and utility controls.
- Must remain clear at small sizes.
- Icon should never carry meaning without a text alternative nearby when context is unclear.

## Floating CTA

- Used on mobile or in high-intent browsing contexts.
- Should not dominate the page.
- Must remain readable, easy to dismiss, and visually consistent with the system.

## Hover

- Slight elevation or tonal shift only.
- Border, fill, or text changes should remain subtle.
- Hover must make the button feel more alive, not more flashy.

## Focus

- Focus states must be highly visible.
- Use a clear ring or outline that works in both light and dark mode.
- Focus should never rely on color alone.

## Disabled

- Disabled buttons must look unambiguous and inert.
- Maintain legibility but reduce prominence.
- Disabled state should be present only when there is a meaningful reason.

## Loading State

- Loading should preserve button size and shape.
- Use calm feedback rather than spinning theatrics.
- Copy may change to indicate processing when helpful.

## Button Rules

- Buttons should have a consistent radius across the system.
- Labels should be concise and action-oriented.
- Avoid more than two primary CTA styles in the entire product.
- Do not introduce playful motion or excessive glow on hover.
