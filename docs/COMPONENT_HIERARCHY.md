# Component Hierarchy

## Level 1: Layout Shell

- Header
- Footer
- Global navigation
- Page transitions
- Announcement or trust bar

## Level 2: Page Sections

- Hero sections
- Category feature bands
- Trust proof bands
- Product grids
- Use-case panels
- Testimonials
- FAQ blocks
- CTA sections

## Level 3: Content Components

- Headings
- Body text blocks
- Tag labels
- Metric cards
- Feature lists
- Quote blocks
- Image panels
- Swiper or carousel modules where necessary

## Level 4: Interaction Components

- Buttons
- Links
- Forms
- Filter chips
- Accordions
- Tabs
- Dialogs or drawers
- Toasts or status messages

## Hierarchy Principle

Every component should exist to serve one of three functions: orientation, trust, or conversion. If a component does not improve one of those functions, it should not exist.
